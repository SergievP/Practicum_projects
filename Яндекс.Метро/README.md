# Тестирование mobile-приложения "Яндекс.Метро"

## О проекте

"Яндекс.Метро" — мобильное приложение для навигации в метрополитене.  
Сервис предоставляет интерактивную схему метро, позволяет строить маршруты между станциями и рассчитывает предполагаемое время в пути.  

Дополнительно приложение отображает актуальные уведомления об изменениях в работе станций и графике движения.

**Цель тестирования:**
Проверка корректности построения маршрутов, расчёта времени в пути и отображения пользовательского интерфейса.


## Структура тестовой документации
Проект разделен  на две части:
- **Часть 1 — валидация полей ввода и расчёта маршрута** 
    + Файл: `Яндек_Маршруты_pt1.xlsx`
- **Часть 2 — бронирование каршеринга и оплата** 
    + Файл: `Яндек_Маршруты_pt2.xlsx`

## Область тестирования
**Часть 1**
- Валидация полей ввода времени начала поездки (часы, минуты)
- Валидация полей ввода адреса (куда, откуда)
- Проверка корректности расчёта стоимости и времени поездки

**Часть 2**
- Отображение формы бронирования и элементов навигации на карте
- Проверка работы окон «Способ оплаты», «Добавление карты»
- Проверка работы кнопки "Забронировать"

## Результаты
- **Часть 1**
    * 7 критических дефектов - валидация форм ввода времени начала поездки и адресов
    * 2 критических дефекта - некорректный расчет стоимости и времени поездки

- **Часть 2**
    * 1 блокирующий дефект - валидация некорректных данных водительских прав
    * 3 критических дефекта - отображение формы бронирования и элементов навигации на карте
    * 3 критических дефекта - некорректная работа окон "Способ оплаты", "Добавление карты" и кнопки "Заборонировать"

- Дефекты средней и низкой приоритетности - отображение UI и обработкой пользовательского ввода

**Всего выявлено: 16 значимых дефектов**

## Примеры найденных дефектов
### Баг №1 — Некорректный расчет стоимости 
- **Описание:** при выборе режима "На своем автомобиле" стоимость поездки рассчитывается некорректно.
- **Ожидаемый результат:** корректный расчёт стоимости.
- **Фактический результат:** стоимость уменьшается примерно в 2 раза.

![Некорректный расчет стоимости](Screenshots/bug-1.png)

### Баг №2 — Некорректное отображение данных забронированного автомобиля
- **Описание:** после успешного бронирования каршерингового автомобиля не отображаются марка и государственный номер машины.
- **Ожидаемый результат:** отображаются корректные данные автомобиля
- **Фактический результат:** отображается только название тарифа.

![Некорректное отображение данных забронированного автомобиля](Screenshots/bug-2.png)
