# Тестирование API приложения "Яндекс.Прилавок"

## О проекте
"Яндекс.Прилавок" — сервис по доставке продуктов, готовой еды.

**Цель тестирования:**
Проверить корректность обработки запросов API, валидацию входных данных и бизнес-логику расчёта доставки.

- **Скриншоты**

## Область тестирования
Тестирование следующих endpoint'ов (ручек):
- `POST /api/v1/kits/{id}/products` — добавление продуктов в набор
- `POST /fast-delivery/v3.1.1/calculate-delivery.xml` — расчёт стоимости доставки
- `GET /api/v1/orders/:id` — получение списка продуктов, добавленных в корзину
- `PUT /api/v1/orders/:id` — добавление продуктов в корзину
- `DELETE/api/v1/orders/:id` - удаление корзины

Проверены:
- авторизация (authToken)
- граничные значения (id, количество, вес)
- негативные сценарии
- корректность HTTP-статусов и структуры ответа

## Результаты
Выявлено 50 дефектов:
- **Блокирующие (7)** - получение продуктов в корзине, ошибки авторизации, невозможность удаления корзины, добавление невалидных товаров в корзину.
- **Критические (6)** - некорректная обработка отрицательных значений, неверный расчёт стоимости доставки, ошибки валидации.
- **Средние (37)** - несоответствие формата ответа, некорректные сообщения об ошибках, проблемы обработки граничных значений.

## Примеры найденных дефектов
### Баг №1 — Некорректный HTTP-статус при добавлении товара
- **Описание:** при отправке `POST /api/v1/orders/1` возвращается `404 Not Found`
- **Ожидаемый результат:** `200 ОК`
- **Фактический результат:** `404 Not Found`

![Некорректный HTTP-статус при добавлении товара](Screenshots/bug-1.png)

### Баг №2 — Некорректный расчет стоимости доставки при граничном значении веса
- **Описание:** при весе заказа от 2,6 кг в доставке возвращается некорректная стоимость доставки
- **Ожидаемый результат:** `hostDeliveryCost="43"`
- **Фактический результат:** `hostDeliveryCost="23"`

![](Screenshots/bug-2.png)


## Вывод

В ходе тестирования выявлены критические дефекты в обработке авторизации и валидации входных данных.  
Основные риски связаны с некорректной обработкой негативных сценариев и граничных значений, что может приводить к ошибкам бизнес-логики и финансовым потерям.
